# AI活用方針

---

## 基本方針

AIをコード自動生成ツールとしてではなく、理解・設計・レビューの支援ツールとして使う。

- AIが生成したコードは必ず自分で読んでから採用する
- 処理の流れと設計理由を自分の言葉で説明できる状態を目指す
- 分からないまま進めない。理解できるまで聞き直す

---

## 使用ツールと役割

| ツール | 用途 |
| --- | --- |
| Cursor | コーディング補助・インライン提案 |
| Claude | 設計相談・コードレビュー・概念説明 |
| ChatGPT | 調査・用語確認・別の視点からの意見 |

---

## AIへの依頼パターン

### 概念を理解したいとき

```
「〇〇とは何ですか？初心者向けに説明してください」
「〇〇と△△の違いを教えてください」
```

### 設計を相談したいとき

```
「〇〇をするとき、AとBの2つの方法があります。
 今の構成ではどちらが適切ですか？理由も教えてください」
```

### コードレビューを依頼するとき

```
「以下のコードをレビューしてください。
 責務分割・データの流れ・改善点があれば指摘してください」
```

---

## AIでの禁止事項

- コードの丸コピー（読んで理解してから使う）
- 日報の代筆（気づきは自分の言葉で書く）
- 設計判断の丸投げ（候補を出してもらい、自分で選ぶ）
