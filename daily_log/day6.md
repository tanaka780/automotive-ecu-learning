## 振り返りで確認した内容

### 基礎理解（C言語）
- struct / typedef / ポインタ / const / enum / static / 固定幅整数型を、
  自分の実装（sensor.c, status.c等）に紐づけて説明できる状態まで確認した。
- 初回の説明で混同していた点（struct と typedef、int と uint8_t の範囲認識など）は、
  指摘を受けてその場で修正し、再言語化した。

### 設計判断の理由を説明できる
- alert.c と status.c を分けた理由（即時警告 vs 状態の段階分類）を、
　「似ているが役割が違う」という観点で説明できた。
- 固定幅整数型の選定理由を、実際の数値範囲から計算して説明できた
  （例: rpm_sum 最大120,000 → uint16_tでは不足 → uint32_t）。
