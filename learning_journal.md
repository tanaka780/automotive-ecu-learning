## C言語基礎

### struct

関連するデータをまとめる型。speed / rpm / temperature を1つにまとめることで、関数に渡すときに1つ渡すだけで済む。

### .h ファイル

他のファイルに関数を公開するための宣言を書く場所。実装（中身）は .c に書く。

### Makefile

ビルドを自動化する仕組み。`make` だけで複数ファイルのコンパイルをまとめて実行できる。

### typedef

`struct VehicleSensorData` と毎回書かずに `VehicleSensorData` だけで使えるようにする仕組み。型に短い別名をつける。

### & とポインタ

- `&sensor_data` は変数に自動で割り当てられた住所（番地）を調べる演算子
- `VehicleSensorData *data` は住所を受け取るための変数（ポインタ）。値ではなく住所そのものが入っている
- 値をそのまま渡すとデータのコピーが作られる。コピーを書き換えても元データは変わらない。住所を渡すと元データを直接書き換えられる

### -> 演算子

住所（ポインタ）の先にある箱の特定のメンバーにアクセスする。読み取りにも書き込みにも使う。
例: `data->speed` → data に入っている住所の先にある speed にアクセスする

### const ポインタ

`const VehicleSensorData *data` はポインタ経由での書き込みを禁止する。
「読むだけ」の責務を型で明示するために使う。

## 設計パターン

### 責務分割

1つのモジュール（.c ファイル）が持つ仕事を1つに限定する。

- `sensor.c`：今この瞬間のセンサ値を管理する
- `stats.c`：これまでの値の最小・最大・合計を管理する
- `alert.c`：閾値チェックと警告出力のみを担当する

混ぜると「このファイルは何をするファイルか」を一言で説明できなくなる。
また、一方を修正したときにもう一方に影響するリスクが生まれる。

### #define マクロ

定数に名前をつける仕組み。コンパイル前にプリプロセッサが数値に置き換える。
`ALERT_SPEED_MAX` のように書くことで、数値の意味がコードから読み取れる。
閾値を変更したいときは .h の定義箇所だけを修正すればよい。

## C言語基礎（追加）

### 整数除算と型変換

- `int ÷ int` はコンパイラが演算時点で int として計算する（小数点以下は切り捨て）
- `float result = int_a / int_b` は int 除算の後に float に変換するため、結果は小数にならない
- float の精度が必要な場合は `(float)int_a / int_b` のようにキャストが必要
- 式内のオペランドの型で計算し、結果を代入先の変数の型に変換して格納する
